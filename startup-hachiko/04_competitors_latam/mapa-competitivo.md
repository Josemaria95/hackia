# Mapa Competitivo — Hachiko en el Landscape

Dos vistas complementarias del espacio competitivo. La primera mapea el enfoque clinico; la segunda mapea la capacidad de gamificacion y enterprise.

---

## Vista 1: Enfoque Clinico vs Enfoque de Datos

Ejes:
- **X:** Reactivo (terapia post-crisis) ←→ Preventivo (antes de que pase)
- **Y:** Clinico/Terapia ←→ Datos/Predictivo

```

                              |
                 Yerbo        |        HACHIKO

                              |         compliance)

      matching)              |

  REACTIVO ───────────────────┼─────────────────── PREVENTIVO

### Analisis por cuadrante

**Cuadrante superior-derecho (Preventivo + Datos):** Es donde Hachiko se posiciona. Hoy esta practicamente vacio. Yerbo se acerca por el lado de analytics de burnout pero no tiene gamificacion ni capa B2C. Spring Health tiene datos pero es reactivo (matching a terapeutas). **Este es el gap principal del mercado.**

**Cuadrante inferior-derecho (Preventivo + Clinico):** Finch, Habitica y Betterfly viven aqui. Ofrecen prevencion via habitos saludables pero no generan datos accionables para empresas. Son B2C puros (Finch, Habitica) o B2B con poca profundidad de datos (Betterfly).

**Cuadrante inferior-izquierdo (Reactivo + Clinico):** El cuadrante mas poblado. EAPs digitales (Modern Health, Momentu, Selia) y chatbots terapeuticos (Woebot, Wysa, Replika). Resuelven el problema cuando ya existe — no lo previenen. Tasas de uso bajas (3-5% para EAPs).

**Cuadrante superior-izquierdo (Reactivo + Datos):** Spring Health ocupa parcialmente este espacio con su enfoque de "precision mental health" — usa datos para optimizar la respuesta reactiva (mejor matching terapeuta-paciente). Pero sigue siendo reactivo.

## Vista 2: Gamificacion vs Capacidad B2B

Ejes:
- **X:** Sin gamificacion ←→ Gamificacion profunda
- **Y:** B2C puro ←→ Enterprise/B2B

                              |         white-label)

      some engagement)        |

                              |

     (AI companion,           |        (mascota virtual,
      light engagement)       |         B2C puro)
                              |
                        B2C PURO
```

### Analisis de la vista 2

**El gap es evidente:** ningun competidor combina gamificacion profunda con capacidad enterprise. Los que tienen gamificacion (Finch, Habitica) son B2C puros. Los que son B2B (Modern Health, Spring Health) no tienen gamificacion.

**Betterfly** es el mas cercano a cerrar el gap, con habitos gamificados y modelo B2B. Pero su gamificacion es superficial (puntos → donaciones) y su foco es seguros, no datos de salud mental.

**Hachiko ocupa el cuadrante superior-derecho:** gamificacion profunda (8 Core Drives, mascota virtual con vinculo emocional) + capacidad enterprise completa (dashboard, risk scores, compliance, white-label).

## Posicion unica de Hachiko

### Diferenciadores combinados

| Diferenciador | ¿Quien mas lo tiene? |
|--------------|---------------------|
| Mascota virtual con vinculo emocional | Finch (pero B2C puro) |
| Gamificacion basada en Octalysis (8 Core Drives) | Nadie en healthtech |
| Instrumentos clinicos camuflados en gameplay | Nadie |
| Dashboard B2B con risk scores | Yerbo (parcial), Spring Health (no LATAM) |
| White-label para mutuales/aseguradoras | Nadie en LATAM |
| Compliance nativo (ISTAS-21, NOM-035) | Nadie de forma integrada |
| Privacy by Design con Ley 21.719 | Nadie aun (ley es nueva) |
| SLM local + LLM cloud (arquitectura hibrida) | Nadie en este segmento |
| Foco LATAM desde el inicio | Betterfly, Momentu, Selia (pero sin datos predictivos) |

### La combinacion que nadie tiene

```
Gamificacion profunda (Finch)
    + Datos predictivos B2B (Spring Health)
    + Compliance regulatorio LATAM (nadie)
    + Canal mutuales (nadie)
    = HACHIKO
```

Cada pieza individual existe en algun competidor. La combinacion completa no existe en ninguno.
